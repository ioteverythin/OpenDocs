Docs/Overview

OpenDocs

Convert GitHub READMEs, npm packages, Markdown files, and Jupyter Notebooks into structured, multi-format documentation — instantly.

🆕 What’s new in v0.9.0

Mindmap output format — generates Mermaid mindmap diagrams + hierarchical JSON from any documentation source. Also: npm: package source support (v0.8) and full CI/CD pipeline.

What is OpenDocs?

OpenDocs (by ioteverythin) takes a GitHub repository README, npm package, local Markdown file, or Jupyter Notebook and automatically generates beautiful, professional documentation in 13 output formats.

It ships two engines: a deterministic pipeline (fast, free, no LLM needed) and DocAgent (a full agentic system for enterprise document generation).

Key Features

📄 13 Output Formats

Word, PDF, PPTX, Blog, Jira, Changelog, LaTeX, One-Pager, Social Cards, FAQ, Architecture, Mindmap — from a single command.

🎨 25 Built-in Themes

Classic, Professional, and Modern theme categories. Corporate, Aurora, Carbon, Graphite, and more.

🧠 5 LLM Providers

OpenAI, Anthropic (Claude), Google (Gemini), Ollama (local), Azure OpenAI — or zero-LLM basic mode.

📦 Multiple Sources

GitHub URLs, npm packages, local .md files, folder merging, and Jupyter Notebooks (.ipynb).

🔍 Knowledge Graph

Extracts 10+ entity types: projects, technologies, APIs, metrics, prerequisites. Generates architecture diagrams.

👀 File Watcher

Monitor repos for changes, auto-regenerate docs, and create pull requests with opendocs watch.

📝 Template Variables

Inject project name, author, version, org into headers/footers via YAML config or CLI flags.

🤖 DocAgent

Agentic engine generating PRD, Proposal, SOP, Report, Slides, Changelog, Onboarding, and Tech Debt docs.

Docs/Get Started/Quickstart

Quickstart

Go from zero to multi-format documentation in under 2 minutes.

✓ Prerequisites

Python ≥ 3.10. Install with pip install opendocs.

1. Install

bash
pip install opendocs

2. Generate docs from a GitHub repo

bash
opendocs generate https://github.com/fastapi/fastapi

This fetches the README and generates all 13 formats into an ./output folder.

3. Pick a specific format and theme

bash
opendocs generate https://github.com/fastapi/fastapi --format word --theme aurora

4. Generate from an npm package

bash
opendocs generate npm:axios --format blog

5. Generate from a local file

bash
opendocs generate ./README.md --local --format pdf

6. Enable LLM mode

bash
opendocs generate ./README.md --local --mode llm --api-key sk-...

LLM mode adds knowledge graph extraction, executive summaries, and stakeholder views (CTO, Investor, Developer).

✓ Next steps

Explore all 13 output formats, try 25 built-in themes, or set up the file watcher.

Docs/Sources/GitHub URLs

GitHub URLs

Point OpenDocs at any public GitHub repository to fetch the README and generate documentation.

Basic usage

bash
opendocs generate https://github.com/owner/repo

OpenDocs extracts the owner and repo from the URL, fetches the default-branch README via the GitHub API (using httpx), and passes it through the full pipeline.

Private repos

bash
opendocs generate https://github.com/owner/private-repo --token ghp_...

Pass a GitHub Personal Access Token with --token or set GITHUB_TOKEN in your environment.

Choose output format

bash
# Just a Word document
opendocs generate https://github.com/owner/repo --format word

# Blog post with a theme
opendocs generate https://github.com/owner/repo --format blog --theme carbon

# All formats (default)
opendocs generate https://github.com/owner/repo --format all

Docs/Sources/npm Packages

npm Packages

Generate documentation directly from any npm package — no clone needed. Added in v0.8.0.

ℹ Available since v0.8

Use the npm: prefix to fetch a package’s README from the npm registry.

Basic usage

bash
opendocs generate npm:axios

Scoped packages

bash
opendocs generate npm:@angular/core

With format and theme

bash
opendocs generate npm:express --format word --theme ocean
opendocs generate npm:react --format blog --theme aurora

How it works

  • Queries the npm registry at https://registry.npmjs.org/<package>
  • Extracts the readme field from the package metadata
  • Strips version specifiers (axios@1.6.0 → axios)
  • Handles scoped packages (@scope/pkg) automatically
  • Links back to https://www.npmjs.com/package/<name>

Docs/Sources/Jupyter Notebooks

Jupyter Notebooks

Generate polished reports from .ipynb research notebooks and data-science projects.

Usage

bash
# Generate all formats from a notebook
opendocs generate ./analysis.ipynb --local

# Word report with a theme, excluding cell outputs
opendocs generate ./research.ipynb --local --format word --theme carbon --no-outputs

What gets extracted

  • Markdown cells — parsed into headings, paragraphs, lists, tables
  • Code cells — preserved with language detection and execution count
  • Cell outputs — text output, images (PNG/SVG/JPEG as data URIs), HTML previews, error tracebacks

Use --no-outputs to exclude cell outputs for a cleaner document.

Outputs .mmd source files, rendered .png images, and a Markdown report summarizing all five views.

Docs/Output Formats/Mindmap

Mindmap

Generate a Mermaid mindmap diagram and hierarchical JSON from any documentation source. Added in v0.9.0.

ℹ Available since v0.9

The mindmap format walks the document’s section tree and produces a visual overview of the entire document.

Usage

bash
opendocs generate https://github.com/owner/repo --format mindmap
opendocs generate npm:axios --format mindmap

Output files

  • mindmap_<name>.md — Mermaid mindmap diagram + section index + embedded JSON
  • mindmap_<name>.json — Hierarchical JSON: { name, description, children[] }

Configuration

The generator respects these limits for clean output:

  • MAX_DEPTH = 4 — maximum nesting depth
  • MAX_LABEL = 48 — maximum characters per node label
  • MAX_LIST_CHILDREN = 5 — maximum list items per node

Docs/Features/Template Variables

Template Variables

Inject project metadata into document headers, footers, and title pages via CLI flags or a YAML config file.

Via CLI flags

bash
opendocs generate ./README.md --local \
  --project-name "My Project" \
  --author "Jane Doe" \
  --doc-version "2.1.0" \
  --org "Acme Corp" \
  --department "Engineering" \
  --confidentiality "Internal"

Via YAML config

bash
opendocs generate ./README.md --local --config opendocs.yaml
yaml — opendocs.yaml
project_name: "My Project"
author: "Jane Doe"
version: "2.1.0"
date: "2026-03-20"
organisation: "Acme Corp"
department: "Engineering"
confidentiality: "Internal"
custom:
  reviewer: "John Smith"
  status: "Draft"

Where variables appear

  • Word (.docx) — document header, footer, and expanded metadata table on title page
  • PowerPoint (.pptx) — title slide footer with org, author, version, and date
  • PDF — inherits from Word generator

Docs/Features/File Watcher

File Watcher & Auto-PR

Monitor a repository for changes and auto-regenerate documentation. Optionally create pull requests automatically.

Usage

bash
# Continuous watch (checks every 30 seconds)
opendocs watch ./my-repo

# One-shot mode for cron jobs
opendocs watch ./my-repo --once

# Watch + auto-create pull requests
opendocs watch ./my-repo --auto-pr --branch docs-update

# Custom interval and file patterns
opendocs watch ./my-repo --interval 60 --patterns "README.md,docs/*.md,*.ipynb"

How it works

  1. Discovers files matching watch patterns (README.md, CHANGELOG.md, docs/**/*.md, *.ipynb)
  2. Computes SHA-256 hashes and compares against saved state (.opendocs-watch-state.json)
  3. If changes detected: runs the full pipeline for each changed file
  4. If --auto-pr: creates a timestamped git branch, commits outputs, pushes, and opens a PR via GitHub CLI (gh)

Cron integration

cron
0 * * * * cd /path/to/repo && opendocs watch . --once --auto-pr

Knowledge Graph

OpenDocs automatically extracts entities and relationships from your documentation to power architecture diagrams, PowerPoint slides, and analysis reports.

Entity types

PROJECT TECHNOLOGY API_ENDPOINT METRIC CONCEPT PREREQUISITE CLOUD_SERVICE DATABASE FRAMEWORK PLATFORM

How it works

  • Semantic extraction (always on) — regex-based entity and relation discovery from document AST
  • LLM extraction (opt-in) — uses the selected LLM provider to discover deeper entities and relationships

Docs/DocAgent/Agentic Engine

DocAgent — Agentic Engine

A full agentic system that clones any GitHub repo and generates 8 enterprise document types using LLM-powered skills.

ℹ Requires LLM

Install with pip install opendocs[agents]. Needs an LLM API key.

Document types

TypeDescription
prdProduct Requirements Document
proposalTechnical Proposal / SOW
sopStandard Operating Procedure
reportTechnical Assessment Report
slidesExecutive Slide Deck
changelogChangelog from Git History
onboardingDeveloper Onboarding Guide
tech-debtTech Debt Assessment

Usage

bash
# Generate a PRD from a GitHub repo
opendocs agent https://github.com/owner/repo --type prd

# Generate all document types
opendocs agent https://github.com/owner/repo --type all

# Use Claude instead of OpenAI
opendocs agent https://github.com/owner/repo --type report --provider anthropic

How it works

  1. Clone — clones the repo locally
  2. Crawl — discovers file tree, key files, dependencies
  3. Index — builds a RepoKnowledgeModel with architecture, tech stack, data flow
  4. Generate — LLM skills produce each document type
  5. Review — quality checks and structure validation
  6. Export — renders to Word, PDF, Markdown

Docs/Reference/Python API

Python API

Use OpenDocs programmatically in your Python applications.

Basic usage

python
from opendocs.pipeline import Pipeline

pipeline = Pipeline()
result = pipeline.run("https://github.com/owner/repo", theme_name="aurora")

Specific formats

python
from opendocs.pipeline import Pipeline
from opendocs.core.models import OutputFormat

pipeline = Pipeline()
result = pipeline.run(
    "./README.md",
    local=True,
    formats=[OutputFormat.WORD, OutputFormat.PDF, OutputFormat.MINDMAP],
)

With template variables

python
from opendocs.pipeline import Pipeline
from opendocs.core.template_vars import TemplateVars

tvars = TemplateVars(
    project_name="Q4 Analysis",
    author="Data Team",
    version="1.0",
    organisation="Acme Corp",
)
pipeline = Pipeline()
pipeline.run("./notebook.ipynb", local=True, template_vars=tvars)

LLM mode

python
pipeline.run(
    "./README.md",
    local=True,
    mode="llm",
    api_key="sk-...",
    provider="anthropic",
)
