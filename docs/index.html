Docs/Overview

OpenDocs

Convert GitHub READMEs, npm packages, Markdown files, and Jupyter Notebooks into structured, multi-format documentation instantly.

🆕 What’s new in v0.9.0

Mindmap output format generates Mermaid mindmap diagrams + hierarchical JSON from any documentation source. Also: npm: package source support (v0.8) and full CI/CD pipeline.

What is OpenDocs?

OpenDocs (by ioteverythin) takes a GitHub repository README, npm package, local Markdown file, or Jupyter Notebook and automatically generates beautiful, professional documentation in 13 output formats.

It ships two engines: a deterministic pipeline (fast, free, no LLM needed) and DocAgent (a full agentic system for enterprise document generation).

Key Features

📄 13 Output Formats

Word, PDF, PPTX, Blog, Jira, Changelog, LaTeX, One-Pager, Social Cards, FAQ, Architecture, Mindmap from a single command.

🎨 25 Built-in Themes

Classic, Professional, and Modern theme categories. Corporate, Aurora, Carbon, Graphite, and more.

🧠 5 LLM Providers

OpenAI, Anthropic (Claude), Google (Gemini), Ollama (local), Azure OpenAI or zero-LLM basic mode.

📦 Multiple Sources

GitHub URLs, npm packages, local .md files, folder merging, and Jupyter Notebooks (.ipynb).

🔍 Knowledge Graph

Extracts 10+ entity types: projects, technologies, APIs, metrics, prerequisites. Generates architecture diagrams.

👀 File Watcher

Monitor repos for changes, auto-regenerate docs, and create pull requests with opendocs watch.

📝 Template Variables

Inject project name, author, version, org into headers/footers via YAML config or CLI flags.

🤖 DocAgent

Agentic engine generating PRD, Proposal, SOP, Report, Slides, Changelog, Onboarding, and Tech Debt docs.

📊 Smart Table Sorting

6 strategies smart, alpha, numeric, column:N, column:N:desc, none reorder tables for cleaner output.

🖼 Mermaid → PNG

Auto-render Mermaid diagram blocks into PNG images embedded in Word, PDF, and PowerPoint.

📤 Publishing

Push generated docs directly to Notion workspaces or Confluence spaces with a single flag.

👥 Stakeholder Views

LLM-powered perspectives for CTO, Investor, and Developer audiences in every output.

🤖 AI Reader Files

Auto-generate llms.txt, llms-full.txt, AGENTS.md, and CLAUDE.md for LLMs and coding agents.

Docs/Get Started/Quickstart

Quickstart

Go from zero to multi-format documentation in under 2 minutes.

✓ Prerequisites

Python ≥ 3.10. Install with pip install opendocs.

1. Install

bash
pip install opendocs

2. Generate docs from a GitHub repo

bash
opendocs generate https://github.com/fastapi/fastapi

This fetches the README and generates all 13 formats into an ./output folder.

3. Pick a specific format and theme

bash
opendocs generate https://github.com/fastapi/fastapi --format word --theme aurora

4. Generate from an npm package

bash
opendocs generate npm:axios --format blog

5. Generate from a local file

bash
opendocs generate ./README.md --local --format pdf

6. Enable LLM mode

bash
opendocs generate ./README.md --local --mode llm --api-key sk-...

LLM mode adds knowledge graph extraction, executive summaries, and stakeholder views (CTO, Investor, Developer).

Docs/Sources/GitHub URLs

GitHub URLs

Point OpenDocs at any public GitHub repository to fetch the README and generate documentation.

Basic usage

bash
opendocs generate https://github.com/owner/repo

OpenDocs extracts the owner and repo from the URL, fetches the default-branch README via the GitHub API (using httpx), and passes it through the full pipeline.

Private repos

bash
opendocs generate https://github.com/owner/private-repo --token ghp_...

Pass a GitHub Personal Access Token with --token or set GITHUB_TOKEN in your environment.

Choose output format

bash
# Just a Word document
opendocs generate https://github.com/owner/repo --format word

# Blog post with a theme
opendocs generate https://github.com/owner/repo --format blog --theme carbon

# All formats (default)
opendocs generate https://github.com/owner/repo --format all

Advanced examples

bash
# LLM-enhanced Word report with template vars
opendocs generate https://github.com/fastapi/fastapi \
  --format word --theme aurora \
  --mode llm --api-key sk-... \
  --project-name "FastAPI Analysis" \
  --author "DevOps Team" --org "Acme Corp"

# Architecture diagrams from a popular framework
opendocs generate https://github.com/django/django --format architecture

# Mindmap of a library's documentation structure
opendocs generate https://github.com/pallets/flask --format mindmap

# PPTX deck for a board presentation
opendocs generate https://github.com/owner/repo \
  --format pptx --theme graphite \
  --mode llm --provider anthropic

# Generate Jira tickets + changelog
opendocs generate https://github.com/owner/repo --format jira
opendocs generate https://github.com/owner/repo --format changelog

# Generate and publish to Notion
opendocs generate https://github.com/owner/repo --format blog \
  --publish notion --notion-token ntn_... --notion-page-id abc123

Python API

python
from opendocs.pipeline import Pipeline

pipeline = Pipeline()

# Basic GitHub repo
result = pipeline.run("https://github.com/fastapi/fastapi")

# With specific formats and theme
from opendocs.core.models import OutputFormat
result = pipeline.run(
    "https://github.com/owner/repo",
    formats=[OutputFormat.WORD, OutputFormat.PPTX],
    theme_name="corporate",
)

# Private repo with token
result = pipeline.run(
    "https://github.com/owner/private-repo",
    github_token="ghp_...",
)

Docs/Sources/Local Files

Local Files & Folders

Generate docs from local Markdown files or merge an entire folder of .md files.

Single file

bash
opendocs generate ./README.md --local

Folder merging

bash
# Merge all .md files in docs/ into one document
opendocs generate ./docs/ --output ./exported

When pointed at a directory, OpenDocs discovers all .md files, merges them in alphabetical order, and runs the full pipeline on the combined content.

More examples

bash
# Generate a themed Word doc from a local README
opendocs generate ./README.md --local --format word --theme carbon

# PDF with custom branding
opendocs generate ./README.md --local --format pdf \
  --theme corporate \
  --project-name "Q4 Report" \
  --author "Data Team" \
  --org "Acme Corp"

# Generate architecture diagrams from a design doc
opendocs generate ./ARCHITECTURE.md --local --format architecture

# Blog post from a changelog
opendocs generate ./CHANGELOG.md --local --format blog --theme aurora

# Merge a whole docs folder into a single PPTX
opendocs generate ./docs/ --format pptx --theme graphite --output ./slides

# LLM-enhanced report from a local file
opendocs generate ./README.md --local \
  --mode llm --provider google \
  --format word --theme slate

Python API

python
from opendocs.pipeline import Pipeline
from opendocs.core.models import OutputFormat

pipeline = Pipeline()

# Single local file → Word + PDF
result = pipeline.run(
    "./README.md",
    local=True,
    formats=[OutputFormat.WORD, OutputFormat.PDF],
    theme_name="aurora",
)

# Folder of docs → merged PPTX
result = pipeline.run(
    "./docs/",
    formats=[OutputFormat.PPTX],
    output_dir="./presentations",
)

Docs/Sources/npm Packages

npm Packages

Generate documentation directly from any npm package no clone needed. Added in v0.8.0.

ℹ Available since v0.8

Use the npm: prefix to fetch a package’s README from the npm registry.

Basic usage

bash
opendocs generate npm:axios

Scoped packages

bash
opendocs generate npm:@angular/core

With format and theme

bash
opendocs generate npm:express --format word --theme ocean
opendocs generate npm:react --format blog --theme aurora

More examples

bash
# Blog + PDF with sunset theme
opendocs generate npm:express --format blog --theme sunset
opendocs generate npm:express --format pdf --theme sunset

# Generate all formats from lodash
opendocs generate npm:lodash --format all --theme ocean

# LLM-enhanced docs from React
opendocs generate npm:react --format word \
  --mode llm --provider openai --api-key sk-...

# Architecture diagrams from Vue
opendocs generate npm:vue --format architecture --theme carbon

Python API

python
from opendocs.pipeline import Pipeline
from opendocs.core.models import OutputFormat

pipeline = Pipeline()

# npm package → Word document
result = pipeline.run(
    "npm:axios",
    formats=[OutputFormat.WORD],
    theme_name="corporate",
)

# Scoped package → Blog + PDF
result = pipeline.run(
    "npm:@angular/core",
    formats=[OutputFormat.BLOG, OutputFormat.PDF],
    theme_name="sunset",
)

# Batch process multiple npm packages
for pkg in ["npm:axios", "npm:express", "npm:react", "npm:vue"]:
    name = pkg.replace("npm:", "")
    pipeline.run(pkg, output_dir=f"./output/{name}")
    print(f"✓ {name}")

How it works

  • Queries the npm registry at https://registry.npmjs.org/<package>
  • Extracts the readme field from the package metadata
  • Strips version specifiers (axios@1.6.0 → axios)
  • Handles scoped packages (@scope/pkg) automatically
  • Links back to https://www.npmjs.com/package/<name>

Docs/Sources/Jupyter Notebooks

Jupyter Notebooks

Generate polished reports from .ipynb research notebooks and data-science projects.

Usage

bash
# Generate all formats from a notebook
opendocs generate ./analysis.ipynb --local

# Word report with a theme, excluding cell outputs
opendocs generate ./research.ipynb --local --format word --theme carbon --no-outputs

What gets extracted

  • Markdown cells parsed into headings, paragraphs, lists, tables
  • Code cells preserved with language detection and execution count
  • Cell outputs text output, images (PNG/SVG/JPEG as data URIs), HTML previews, error tracebacks

Use --no-outputs to exclude cell outputs for a cleaner document.

More examples

bash
# PDF report from a data science notebook
opendocs generate ./analysis.ipynb --local --format pdf --theme slate

# Executive PPTX from a research notebook
opendocs generate ./research.ipynb --local --format pptx --theme corporate

# Blog post from a tutorial notebook
opendocs generate ./tutorial.ipynb --local --format blog --theme ocean

# LaTeX academic paper from notebook
opendocs generate ./experiment.ipynb --local --format latex --theme minimal

# Full suite with custom branding
opendocs generate ./quarterly_analysis.ipynb --local \
  --format all --theme aurora \
  --project-name "Q4 2025 Analysis" \
  --author "Data Science Team" \
  --org "Acme Corp"

# LLM-enhanced notebook report
opendocs generate ./ml_pipeline.ipynb --local \
  --mode llm --provider openai \
  --format word --no-outputs

Python API

python
from opendocs.pipeline import Pipeline
from opendocs.core.models import OutputFormat

pipeline = Pipeline()

# Convert notebook to Word report
result = pipeline.run(
    "./analysis.ipynb",
    local=True,
    formats=[OutputFormat.WORD, OutputFormat.PDF],
    theme_name="carbon",
    no_outputs=True,  # Exclude cell outputs
)

# Batch convert all notebooks in a folder
from pathlib import Path
for nb in Path(".").glob("*.ipynb"):
    pipeline.run(str(nb), local=True, formats=[OutputFormat.PDF])

Architecture Diagrams

Auto-generate 5 architectural views from the knowledge graph as Mermaid diagrams rendered to PNG.

The 5 views

  • System Architecture C4-style component diagram
  • Tech Stack Layers horizontal layer diagram
  • Data Flow how data moves through the system
  • Dependency Tree package/module dependencies
  • Deployment View infrastructure and hosting

CLI examples

bash
# From a GitHub repo
opendocs generate https://github.com/fastapi/fastapi --format architecture

opendocs generate npm:express --format architecture

  --format architecture --format word --theme slate

For each source, you get:

  • architecture_system.mmd + .png System architecture (C4-style)
  • architecture_tech_stack.mmd + .png Tech stack layers
  • architecture_data_flow.mmd + .png Data flow
  • architecture_dependencies.mmd + .png Dependency tree

Docs/Output Formats/Mindmap

Mindmap

Generate a Mermaid mindmap diagram and hierarchical JSON from any documentation source. Added in v0.9.0.

ℹ Available since v0.9

The mindmap format walks the document’s section tree and produces a visual overview of the entire document.

Usage

bash
opendocs generate https://github.com/owner/repo --format mindmap
opendocs generate npm:axios --format mindmap

Output files

  • mindmap_<name>.md Mermaid mindmap diagram + section index + embedded JSON
  • mindmap_<name>.json Hierarchical JSON: { name, description, children[] }

Configuration

The generator respects these limits for clean output:

  • MAX_DEPTH = 4 maximum nesting depth
  • MAX_LABEL = 48 maximum characters per node label
  • MAX_LIST_CHILDREN = 5 maximum list items per node

More examples

bash
# Mindmap from a GitHub repo
opendocs generate https://github.com/django/django --format mindmap

# Mindmap from a local README
opendocs generate ./README.md --local --format mindmap

# Mindmap from a Jupyter notebook
opendocs generate ./analysis.ipynb --local --format mindmap

# Mindmap + architecture diagrams together
opendocs generate npm:express --format mindmap --format architecture

# Mindmap + Word report
opendocs generate ./docs/ --format mindmap --format word --theme aurora

Python API

python
from opendocs.pipeline import Pipeline
from opendocs.core.models import OutputFormat

    output_dir="./mindmaps",

# Combine mindmap with other formats

Docs/Features/Template Variables

Template Variables

Inject project metadata into document headers, footers, and title pages via CLI flags or a YAML config file.

Via CLI flags

bash
opendocs generate ./README.md --local \
  --project-name "My Project" \
  --author "Jane Doe" \
  --doc-version "2.1.0" \
  --org "Acme Corp" \
  --department "Engineering" \
  --confidentiality "Internal"

Via YAML config

bash
opendocs generate ./README.md --local --config opendocs.yaml
yaml opendocs.yaml
project_name: "My Project"
author: "Jane Doe"
version: "2.1.0"
date: "2026-03-20"
organisation: "Acme Corp"
department: "Engineering"
confidentiality: "Internal"
custom:
  reviewer: "John Smith"
  status: "Draft"

Where variables appear

  • Word (.docx) document header, footer, and expanded metadata table on title page
  • PowerPoint (.pptx) title slide footer with org, author, version, and date
  • PDF inherits from Word generator

Docs/Features/File Watcher

File Watcher & Auto-PR

Monitor a repository for changes and auto-regenerate documentation. Optionally create pull requests automatically.

Usage

bash
# Continuous watch (checks every 30 seconds)
opendocs watch ./my-repo

# One-shot mode for cron jobs
opendocs watch ./my-repo --once

# Watch + auto-create pull requests
opendocs watch ./my-repo --auto-pr --branch docs-update

# Custom interval and file patterns
opendocs watch ./my-repo --interval 60 --patterns "README.md,docs/*.md,*.ipynb"

How it works

  1. Discovers files matching watch patterns (README.md, CHANGELOG.md, docs/**/*.md, *.ipynb)
  2. Computes SHA-256 hashes and compares against saved state (.opendocs-watch-state.json)
  3. If changes detected: runs the full pipeline for each changed file
  4. If --auto-pr: creates a timestamped git branch, commits outputs, pushes, and opens a PR via GitHub CLI (gh)

Docs/Features/Smart Table Sorting

Smart Table Sorting

Automatically reorder Markdown tables using 6 built-in strategies for cleaner, more readable output.

6 Sorting Strategies

StrategyDescriptionFlag
smartAuto-detect column types and sort by best fit--sort smart
alphaAlphabetical sort on first column--sort alpha
numericNumerical sort (detects numbers in cells)--sort numeric
column:NSort ascending by column N (0-indexed)--sort column:2
column:N:descSort descending by column N--sort column:1:desc
nonePreserve original order--sort none

CLI examples

bash
# Smart sort (default) auto-detect best ordering
opendocs generate ./README.md --local --sort smart

# Sort all tables alphabetically
opendocs generate https://github.com/owner/repo --sort alpha

# Sort by the 3rd column, descending
opendocs generate ./README.md --local --sort column:2:desc

# Disable sorting keep original order
opendocs generate ./README.md --local --sort none

Python API

python
from opendocs.pipeline import Pipeline

pipeline = Pipeline()

# Sort by column 1 descending
pipeline.run("./README.md", local=True, sort_strategy="column:1:desc")

# Numeric sort for benchmark tables
pipeline.run("./benchmarks.md", local=True, sort_strategy="numeric")

How it works

  1. The parser extracts all Markdown tables from the AST
  2. The smart strategy inspects each column for data types (numbers, dates, strings)
  3. Rows are reordered in-place before being passed to generators
  4. The sorted table appears consistently across Word, PDF, PPTX, and LaTeX outputs

Docs/Features/Mermaid → PNG

Mermaid → PNG Rendering

Automatically convert Mermaid diagram code blocks into rendered PNG images embedded in your documents.

Automatic rendering

Any fenced ```mermaid code block in your Markdown is automatically detected, sent to the mermaid.ink API, and the resulting PNG is embedded in Word, PDF, and PowerPoint outputs.

CLI examples

bash
# Any README with ```mermaid blocks gets auto-rendered
opendocs generate https://github.com/owner/repo --format word

# Architecture diagrams are generated as Mermaid and rendered to PNG
opendocs generate ./README.md --local --format architecture

# Mindmaps are also generated as Mermaid syntax
opendocs generate npm:axios --format mindmap

Supported diagram types

Flowchart Sequence Class State ER Diagram Gantt Pie Chart Mindmap C4 Diagram Git Graph

Example input

markdown
```mermaid
graph TD
    A[README.md] --> B[Markdown Parser]
    B --> C[Knowledge Graph]
    C --> D[Word Generator]
    C --> E[PDF Generator]
    C --> F[PPTX Generator]
```

This is automatically converted to a PNG image and embedded in all document outputs.

Python API

python
from opendocs.renderers.diagram_renderer import render_mermaid

# Render a Mermaid diagram to PNG bytes
mermaid_code = """graph TD
    A[Source] --> B[Parser]
    B --> C[Generator]
"""
png_bytes = render_mermaid(mermaid_code)

# Save to file
with open("diagram.png", "wb") as f:
    f.write(png_bytes)

Docs/Reference/Architecture

Pipeline Architecture

How OpenDocs processes sources and generates documents through the deterministic pipeline.

Pipeline flow

GitHub URL / npm:pkg / Local .md / .ipynb
        |
        v
+-------------------+
|  README Fetch /   |  <-- httpx + GitHub API / npm registry
|  Notebook Parser  |  <-- .ipynb cell extraction
+--------+----------+
         v
+-------------------+
|  Markdown Parser  |  <-- mistune 3.x AST
+--------+----------+
         v
+-------------------+
|  Template Vars    |  <-- YAML/JSON config or CLI flags
+--------+----------+
         v
+-------------------+
|  Table Sorting    |  <-- 6 strategies (smart, alpha, numeric, column:N)
+--------+----------+
         v
+-------------------+
|  KG Extraction    |  <-- Semantic + optional LLM (5 providers)
+--------+----------+
         v
+-------------------+
|  Diagram Renderer |  <-- mermaid.ink API
+--------+----------+
         |
    +----+----+----+----+------+------+-------+------+-----+------+------+------+
    v    v    v    v    v      v      v       v      v     v      v      v
  Word  PDF  PPTX  Blog  Jira  Change  LaTeX  1-Pgr  Social  FAQ  Arch  Mindmap
                                 log

Pipeline stages

StageModuleDescription
1. Fetchopendocs.fetchersDownloads README from GitHub API, npm registry, or reads local files/notebooks
2. Parseopendocs.parsersConverts Markdown/Notebook to AST using mistune 3.x
3. Templateopendocs.core.template_varsInjects project metadata (name, author, org, version) into document context
4. Sortopendocs.processors.table_sorterReorders table rows using selected sort strategy
5. Extractopendocs.extractorsBuilds knowledge graph from entities and relationships
6. Renderopendocs.renderersConverts Mermaid diagrams to PNG via mermaid.ink
7. Generateopendocs.generatorsProduces output files in each requested format

File watcher flow

opendocs watch ./repo
        |
        v
  Discover watched files (README.md, *.ipynb, docs/)
        |
        v
  SHA-256 hash each file
        |
        v
  Compare against .opendocs-watch-state.json
        |
        v
  If changed --> Pipeline.run() for each file
        |
        v
  Update state file
        |
        v
  If --auto-pr --> git branch + commit + push + gh pr create

Core dependencies

mistune 3.x python-docx python-pptx httpx Pillow PyYAML nbformat Jinja2

AI Reader Files

Every pipeline run automatically generates four companion files that make your documentation instantly discoverable by LLMs, coding agents, and AI assistants.

✨ Zero configuration

AI reader files are generated automatically no CLI flags or settings needed. They appear alongside your other output files.

Generated Files

FilePurposeSpec
llms.txtCurated index of all generated docsllmstxt.org
llms-full.txtAll content concatenated into one file-
AGENTS.mdCross-agent coding instructionsCodex, Devin, Copilot
CLAUDE.mdClaude Code specific contextClaude Code

llms.txt

Follows the llmstxt.org specification exactly. Provides a structured index of all generated documentation files so LLMs can discover what’s available without reading every file.

# MyProject

> Documentation generated by OpenDocs for MyProject.

## Documentation

- [Word Document](MyProject.docx): Full documentation exported as .docx
- [PDF Document](MyProject.pdf): Printable documentation in PDF format
- [Blog Post](blog_MyProject.md): SEO-friendly Markdown blog post
- [FAQ](MyProject_faq.md): Frequently-asked-questions document

## Optional

- [Full Documentation](llms-full.txt): All documentation in one file

llms-full.txt

Concatenates all generated outputs into a single file. Text-based formats (.md, .tex, .txt) are inlined; binary formats (.docx, .pdf, .pptx) are referenced by filename.

AGENTS.md

Provides context for AI coding agents (Codex, Devin, Copilot Workspace) working in a repository. Includes project metadata, a list of available documentation files, and pointers to companion files.

CLAUDE.md

Same structure as AGENTS.md but adds a “For Claude” section that points to llms-full.txt for full project context, lists any Mermaid diagram files, and references the original source input.

Python API

AI reader files are generated automatically. After Pipeline.run() completes, the results include a list of generated AI reader file paths:

from opendocs import Pipeline

pipeline = Pipeline()
result = pipeline.run("https://github.com/owner/repo")

# AI reader files are in the output directory
for f in result.ai_reader_files:
    print(f"Generated: {f.name}")
